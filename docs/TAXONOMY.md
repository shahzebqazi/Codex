# Research taxonomy

This document organizes findings and artifacts from AI scaffold iterations into a reusable taxonomy. Use it to classify content as you add prompts, components, and workflows from your `.ai` trees and other sources. Cross-link entries to source files and branch decisions where relevant.

## How to use this taxonomy

- **Intake:** When the AI intake agent asks you to add files, place them in your chosen operation directory (e.g. `.ai/inbox/` or `.ai/prompts/`). See [CONTRIBUTING — AI intake agent workflow](../CONTRIBUTING.md#ai-intake-agent-workflow).
- **Classification:** After ingestion, add entries below under the appropriate bucket and link to the source path or artifact.
- **Branches:** Note which branch or tag a finding relates to when it affects structure or release (e.g. `development`, `Production`).
